L3 (20%) - Due: 11/10 - Extended Due Date: 70% 11/24, 50% 12/22, 2017
  • At least the following features should be available (both React-Native and Native):
    • The input form, on submit the collected data should be send using an email, eg. with gmail app, using an intent.
    • An activity where a list of items is presented, when selecting an item a new screen will allow the user to view and edit the item details.
  • No persistence is needed yet.
L5 (30%) Due: 12/8 - Extended Due Date: 70% 12/22, 50% 01/18 2018
  • Demo the following features:
    • CRUD user interfaces.
    • Offline support (sync adapters).
    • Persist data on the local storage.
    • Display a chart when presenting item details.
    • Use dialogs and pickers.
  • Both in React-Native and Android/iOS
L7 (40%) - Due: 01/12 or 01/18 2018 (depending on the lab frequency)
  • Demo the final app, that contains everything mentioned above plus the following:
    • Online support.
    • Implement CRUD REST client operations.
    • Notifications (gcm, ws, etc.)
    • Synchronize the content with a remote location.
    • Authentication (jwt, oauth).
    • Submit data on background threads or/using promises.
    • Navigates between screens/views.
    • Manage the application state outside the user interface layer.
    • Use the observer pattern to notify the user interface about the changes occurred internally.
    • For extra points: Use different layouts, depending on the device size (phone, tablet, etc.).
  • Both in React-Native and Android/iOS
