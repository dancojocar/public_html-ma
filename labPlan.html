Assessment Lab Plan

L1 - Project Details
  • Due: second laboratory
  • See the requirements from the github classroom project associated with your group.
L2 - CRUD Project
  • Assignment plan:
    • Final due: last laboratory before winter holiday
    • Per functionality:
      • The application without local persistence and remote access - due: first laboratory from November, for 100% of the grade. Last laboratory from November, for 70% of the project grade.
      • Local persistence (using an db engine, no files or settings are allowed!) - due: last laboratory from November, for 100% of the grade. First laboratory from December, for 70% of the project grade.
      • Server integration - due: first laboratory from December, for 100% of the grade. Last laboratory from December, for 70% of the project grade.
    • NOTE: The deadlines are final! After the last laboratory, from December, we are not grading L2 results anymore!
    • In order to meet the deadlines, you need to ensure that the last commit, for the work, was done in the previous day of the laboratory day!
      Eg. If your last laboratory in November is on Monday November 26th, then the commit date should be before Monday November 26th 00:00 hours (GMT+2).
  • Implement in BOTH:
    • Native (iOs(Swift or Obj-C) or Android(Java or Kotlin))
    • Non-native (Flutter or React-Native or NativeScript or Any other mobile multi-platform framework)
  • The application should offer CRUD operations on the business objects. The main screen should be a list of the business objects and it should allow at least the following operations:
    • Create - create new elements that will be added to the list.
    • Read - present the elements in a list.
    • Update - allow the user to update an element found in the list.
    • Delete - delete existing elements from the list.
  • The application should persist the content locally, to be able to allow offline access. And while online it should synchronize the content with a remote server (implemented also by you, in the language of your choice, NO server-less solutions are allowed yet). The communication with the server should be managed using REST services, and it should have counterpart operations for each of the operations mentioned above. While offline the application will perform the following actions:
    • On update and delete - a message will be displayed, that the application is offline and the operation is not available.
    • On read - the content from the local database will be displayed.
    • On create - the input will be persisted in the local database and when the application will detect that the device is able to connect to the server will push all the created entries, while the device was offline.
L2 - Bonus Project
  • Due: last laboratory from January
  • Will be a continuation of the L2 project but ONLY on one platform (from: Swift, Obj-C, Java, Kotlin, Flutter, React-Native, Native-Scripts).
  • Will receive extra bonus points for:
    • +1 point for supporting external authentication providers: google, facebook, etc. (firebase-auth is OK).
    • +1 point for using multiple user roles.
    • +2 points for using server-less persistence (like: firebase-database/realtime/firestore).
    • +1 point for using animations.
    • +1 point at least a unit test for one of the CRUD operations.
    • +1 point the app was published on Google Play.
    • +1 point the app was published on Apple Store.
    • +2 points the app has at least 50 users on Google Play + Apple Store.
    • +1 point for using RxJava/RxSwift/RxKotlin
    • +1 point for using Material Design
    • +1 point for using a bug-tracking system, like: firebase analytics, bugsnag
    • +1 point for using AdMob or something similar
    • +1 point for having in-app purchases.
    • +1 point for using a NavigationDrawer.
    • +1 point for giving the user the choice to toggle some app settings.
Final grade
  • MP(L2 - Main Project Grade) = (L2Native + L2Non-native) / 2
  • LG(Laboratory Grade) = if(MP >= 4.5) min(10, L1 * 0.1 + MP * 0.6 + L3 * 0.3) else 0
  • EG(Exam Grade) - the grade from the practical exam
  • FG(Final Grade) = if(LG >= 4.5 && EG >= 5 ) LG * 0.6 + EG * 0.4 else 0
  • Note: In order to take the practical exam you need to have at least 90% of the laboratory presences.
  • If LG < 4.5 or the presences criteria are not meet, than you are allowed to attend only the practical exam in the re-examination session. For this case, the practical exam will have a different requirement, to cover the laboratory grade too.
