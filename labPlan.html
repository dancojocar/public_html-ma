Notes:
  • The deadlines are final! Please note that we are obeying the faculty subgroups!
    Even if you are attending a laboratory with another subgroup the deadlines are the one associated with your faculty subgroup.
    Please check the subgroup allocation with your laboratory instructor.
  • In order to meet the deadlines, you need to ensure that the last commit, for the work, was done on the previous day of the laboratory day!
    Eg. If your last laboratory is on Monday, November 9th, then the commit date should be before November 9th 00:00 hours (GMT+2).
  • The application should offer CRUD operations on business objects. The main screen should be a list of the business objects and it should allow at least the following operations:
    • Create - create new elements that will be added to the list.
    • Read - present the elements in a list.
    • Update - allow the user to update an element found in the list.
    • Delete - delete existing elements from the list.
  • The application should persist the content locally, to be able to allow offline access. And while online it should synchronize the content with a remote server (implemented also by you, in the language of your choice, NO server-less solutions are allowed). The communication with the server should be managed using REST services, and it should have counterpart operations for each of the operations mentioned above. While offline the application will perform the following actions:
    • On update and delete - a message will be displayed, that the application is offline and the operation is not available.
    • On read - the content from the local database will be displayed, with a note that the server connection is down.
    • On create - the input will be persisted in the local database and when the application will detect that the device is able to connect again to the server will push only the created entries, while the device was offline.
    Final grade
    • For meeting all the deadlines and requirements you will gain 2 points for each.
    • LG(Laboratory Grade) = ProjectIdea (2p) + UI-Only-Native (2p) + UI-Only-NonNative (2p) + DB (2p) + Server (2p).
    • EG(Exam Grade) - the grade from the practical exam.
    • FG(Final Grade) = if(LG >= 4.5 && EG >= 5 ) LG * 0.6 + EG * 0.4 else 0
    • In order to take the practical exam, you need to have at least 90% of the laboratory presences.
    • If LG < 4.5 or the criteria of the presence are not meet then you are allowed to attend only the practical exam in the re-examination session. In this case, the practical exam will have a different requirement, to cover the laboratory-grade too.
