Learning outcomes
  • Use a domain-driven layered architecture (activity/fragment/component, model/content, manager/service, net)
  • Put the business logic into a facade stateful layer (service, manager)
  • Use a net layer for REST clients
  • When fetching data, add progress indicators and take into account cancellation techniques
  • Add a pagination mechanism on the REST client, and add also a pagination strategy at the user interface layer (e.g. use infinite scrolling)
